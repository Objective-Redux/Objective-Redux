Best Practices

1. Avoid having complex actions in StateControllers

When should you not have complex actions in a StateController?

While a StateController can be used to create complex actions (e.g. sagas), this can make forming a mental map of actions very difficult.

Consider, for example, a StateController with a complex action that triggers actions in other controllers. Then, imagine that those controllers, in turn, call other StateController instances. This makes the code very hard to follow.

A better practice is to keep complex actions in a StatelessController. The StatelessController can call the various StateControllers and be independent of any particular state.

When is it acceptable to have complex actions in a StateController?

If there are complex actions that do not require another Controller, it might be acceptable to put the complex action in the StateController.

For example, if you have a StateController that gets or sends data to an external API and does not rely on another Controller.
